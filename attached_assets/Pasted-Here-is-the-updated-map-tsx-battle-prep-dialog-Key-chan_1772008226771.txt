First, add two helper functions **at the top of your map.tsx file** (outside the component), so you don't repeat the same logic inline:

```ts
// ── helpers (put above the component) ───────────────────────────────────────

function calcHIT(level: number, dex: number, luk: number): number {
  return 175 + level + dex + Math.floor(luk / 3);
}

function calcFLEE(level: number, agi: number, luk: number): number {
  const fleeA = 100 + level + agi + Math.floor(luk / 5);
  const perfectDodge = Math.floor(luk / 10);
  return fleeA + perfectDodge;
}

function calcHitChance(attackerHIT: number, defenderFLEE: number): number {
  return Math.max(5, Math.min(95, attackerHIT - defenderFLEE));
}
```

### Summary of changes for IT

2. **HIT/FLEE formulas updated** to Classic-RO style everywhere:

   - `FLEE = 100 + level + AGI + floor(LUK/5) + floor(LUK/10)`

3. **Enemy HIT/FLEE** now reads directly from backend fields `e?.hit` and `e?.flee` (set by your updated `generateEnemyStats`), with formula fallback if those fields are missing.

4. **New HIT/FLEE value row** added to all cards (player, companions, enemy) showing raw numbers in green/yellow.

5. **Hit% and Dodge%** renamed to `HIT%` / `DODGE%` for clarity, and computed correctly using the shared helpers.

6. **Enemy HIT%/DODGE%** shown vs main player stats (not per companion), with a small italic note clarifying this.
