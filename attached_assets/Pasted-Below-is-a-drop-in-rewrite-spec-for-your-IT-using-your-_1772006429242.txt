2. HIT / FLEE – UPDATE calculateHitChance
--------------------------------

Use Classic‑style formulas for HIT and FLEE instead of your current `100 + level*2 + dex*1.5`.

### 2.1. Derive HIT/FLEE if not precomputed

In `runTurnBasedCombat` we will set `hit` and `flee` using RO‑style formulas (see section 5). Here we just consume them.

### 2.2. New calculateHitChance

```ts
export function calculateHitChance(attacker: CombatUnit, defender: CombatUnit): number {
  // HIT and FLEE are treated as already computed Classic-RO style values:
  // HIT = 175 + BaseLv + DEX + floor(LUK / 3)
  // FLEE = (100 + BaseLv + AGI + floor(LUK / 5)) + PerfectDodge
  // (PerfectDodge itself is handled separately if you later want that.)

  const attackerHit = attacker.hit ?? 0;
  const defenderFlee = defender.flee ?? 0;

  // Classic-style: HitChance% = clamp(HIT - FLEE, 5, 95)
  let hitChance = attackerHit - defenderFlee;
  if (hitChance < 5) hitChance = 5;
  if (hitChance > 95) hitChance = 95;

  return hitChance;
}
```

If you later want **Perfect Dodge**:

- Add `perfectDodge` to `CombatUnit`.
- Roll it before calling `calculateHitChance`.

3. STATUS ATK + WEAPON VARIANCE
--------------------------------

Create helper functions to compute **StatusATK** and **TotalATK per hit**.

```ts
function getStatusATK(attacker: CombatUnit, isRanged: boolean): number {
  const lv = (attacker as any).level ?? 1;
  const STR = attacker.str ?? 1;
  const AGI = attacker.agi ?? 1;
  const VIT = attacker.vit ?? 1;
  const INT = attacker.int ?? 1;
  const DEX = attacker.dex ?? 1;
  const LUK = attacker.luk ?? 1;

  if (!isRanged) {
    // Melee
    return Math.floor(lv / 4)
         + STR
         + Math.floor(DEX / 5)
         + Math.floor(LUK / 3);
  } else {
    // Ranged
    return Math.floor(lv / 4)
         + Math.floor(STR / 5)
         + DEX
         + Math.floor(LUK / 3);
  }
}

function getWeaponRoll(attacker: CombatUnit): number {
  const baseWeaponATK = attacker.weaponATK ?? attacker.attack ?? 0;
  const weaponLevel = attacker.weaponLevel ?? 1;

  const varianceRange = 0.05 * weaponLevel * baseWeaponATK;
  const variance = (Math.random() * 2 * varianceRange) - varianceRange; // −range ~ +range

  const refinementBonus = attacker.refinementBonus ?? 0;
  const bonusATK = attacker.bonusATK ?? 0;

  return Math.floor(baseWeaponATK + variance + refinementBonus + bonusATK);
}

function getTotalATK(attacker: CombatUnit, isRanged: boolean): number {
  const statusATK = getStatusATK(attacker, isRanged);
  const weaponRoll = getWeaponRoll(attacker);

  return statusATK + weaponRoll;
}
```

- For now you can choose `isRanged` by checking weapon type (you might need a `weaponType` field later). For testing, assume `isRanged = false` except for archers.

4. NEW DAMAGE FUNCTION – REPLACE calculateDamage
--------------------------------

Replace the current `calculateDamage` with a Classic‑style pipeline that still honors your CritDamage, Endowment, and Guard.

```ts
export function calculateDamage(
  attacker: CombatUnit,
  defender: CombatUnit,
  isCritical: boolean = false,
  isRanged: boolean = false
) {
  // 1) Compute TotalATK (RawDamage)
  const rawATK = getTotalATK(attacker, isRanged);
  let damage = rawATK;

  // 2) Apply Hard DEF (armor-based % reduction)
  const hardDEF = defender.hardDEF ?? defender.defense ?? 0;
  const hardFactor = 100 / (100 + Math.max(0, hardDEF)); // avoid negative/0
  damage = Math.floor(damage * hardFactor);

  // 3) Apply Soft DEF (flat)
  const VIT = defender.vit ?? 1;
  const AGI = defender.agi ?? 1;
  const softFromStats = Math.floor(VIT / 2) + Math.floor(AGI / 5);
  const extraSoft = defender.softDEF ?? 0;
  const softDEF = softFromStats + extraSoft;

  damage = damage - softDEF;
  if (damage < 1) damage = 1;

  // 4) Critical modifier (keep your existing behaviour)
  if (isCritical) {
    const critBonus = attacker.critDamage ?? 0; // % from equipment
    const critMult = 1.5 + (critBonus / 100);
    damage = Math.floor(damage * critMult);
  }

  // 5) Endowment (damage reduction)
  const endowmentPoints = Number((defender as any).endowmentPoints) || 0;
  const damageReduction = Math.min(0.35, (endowmentPoints * 0.5) / 100); // max 35%
  damage = Math.floor(damage * (1 - damageReduction));

  // 6) Guard modifier
  if (defender.isGuarding) {
    damage = Math.floor(damage * 0.7);
  }

  // 7) Minimum damage
  if (damage < 1) damage = 1;

  return damage;
}
```

This completely replaces:

```ts

// ...
const defenderDefense = Math.max(1, Number(defender.defense) || 1);
let damage = Math.floor(baseDamage * (baseDamage / defenderDefense) * critMod * (1 - damageReduction));
```

and instead uses TotalATK → HardDEF → SoftDEF.
