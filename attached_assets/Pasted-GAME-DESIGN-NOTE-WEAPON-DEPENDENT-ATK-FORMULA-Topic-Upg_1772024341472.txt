GAME DESIGN NOTE – WEAPON-DEPENDENT ATK FORMULA
Topic: Upgrade StatusATK calculation to be weapon-type aware
Scope: server/combat.ts AND any stat display / level-up screen
Priority: Medium – required before balancing melee vs ranged classes

--------------------------------
0. BACKGROUND
--------------------------------

Currently the game uses a single ATK formula regardless of what weapon the player holds:

  StatusATK = STR + floor(DEX / 5) + floor(LUK / 3)

This is partially correct for melee weapons, but completely wrong for ranged weapons (bow, gun, etc.).

In Classic RO style, the rule is:

- Melee weapons: STR is the primary ATK stat. DEX contributes weakly.
- Ranged weapons: DEX is the primary ATK stat. STR contributes weakly.

We need to apply this distinction in TWO places:

1) Combat (server/combat.ts) – when computing damage per hit.
2) Stat screen / level-up UI – when displaying "your ATK" to the player.

--------------------------------
1. REQUIRED FIELD – weaponType
--------------------------------

Every player, companion, and enemy CombatUnit must expose a weaponType field.

Add to CombatUnit interface:

  weaponType?: WeaponType;

Define WeaponType (add once, reuse everywhere):

```ts
type WeaponType =
  | "dagger"
  | "sword"
  | "twoHandSword"
  | "axe"
  | "mace"
  | "spear"
  | "knuckle"
  | "katar"
  | "book"
  | "staff"
  | "bow"
  | "gun"
  | "instrument"
  | "whip"
  | "none"; // bare hands / unequipped
```

Add two classifier helpers (add once in a shared util file or at top of combat.ts):

```ts
function isMeleeWeapon(type?: WeaponType): boolean {
  return ![
    "bow", "gun", "instrument", "whip"
  ].includes(type ?? "none");
}

function isRangedWeapon(type?: WeaponType): boolean {
  return [
    "bow", "gun", "instrument", "whip"
  ].includes(type ?? "none");
}
```

Default rule: if weaponType is missing or null, treat as melee.

--------------------------------
2. UPDATED STATUSATK FORMULA (combat.ts)
--------------------------------

Replace any current StatusATK calculation with this function:

```ts
function getStatusATK(unit: CombatUnit): number {
  const lv  = (unit as any).level ?? 1;
  const STR = (unit as any).str   ?? 1;
  const DEX = (unit as any).dex   ?? 1;
  const LUK = (unit as any).luk   ?? 1;

  if (isRangedWeapon((unit as any).weaponType)) {
    // RANGED: DEX is primary, STR contributes weakly
    return Math.floor(lv / 4)
         + Math.floor(STR / 5)
         + DEX
         + Math.floor(LUK / 3);
  }

  // MELEE (default): STR is primary, DEX contributes weakly
  return Math.floor(lv / 4)
       + STR
       + Math.floor(DEX / 5)
       + Math.floor(LUK / 3);
}
```

Important: call getStatusATK(unit) in getTotalATK(unit) like this:

```ts
function getTotalATK(unit: CombatUnit): number {
  const statusATK  = getStatusATK(unit);         // weapon-aware
  const weaponRoll = getWeaponRoll(unit);         // weapon variance
  return statusATK + weaponRoll;
}
```

And use getTotalATK(unit) as RawDamage at the start of calculateDamage.

--------------------------------
3. UPDATED WEAPON ATK BONUS (combat.ts)
--------------------------------

When applying the stat % bonus to base weapon ATK, also branch by weapon type:

MELEE: STR gives 0.5% bonus per point on base weapon ATK.
RANGED: DEX gives 0.5% bonus per point on base weapon ATK.

```ts
function getWeaponATKEffective(unit: CombatUnit): number {
  const baseWeaponATK = (unit as any).weaponATK ?? unit.attack ?? 0;
  const STR = (unit as any).str ?? 1;
  const DEX = (unit as any).dex ?? 1;

  if (isRangedWeapon((unit as any).weaponType)) {
    return Math.floor(baseWeaponATK * (1 + 0.005 * DEX));
  }

  return Math.floor(baseWeaponATK * (1 + 0.005 * STR));
}
```

--------------------------------
4. STAT DISPLAY AND LEVEL-UP SCREEN
--------------------------------

The stat screen (wherever ATK is shown to the player) must ALSO use this weapon-aware formula. Currently if the stat screen shows a fixed ATK number, it may show the wrong value for ranged players.

Changes needed in the frontend stat display component:

4.1. Pass weaponType to the ATK display calculation.

When computing "ATK shown on stat screen":

```ts
function displayATK(player: PlayerStats): number {
  const lv  = player.level ?? 1;
  const STR = player.str ?? 1;
  const DEX = player.dex ?? 1;
  const LUK = player.luk ?? 1;
  const weaponType = (player as any).weaponType;

  let statusATK: number;

  if (isRangedWeapon(weaponType)) {
    statusATK = Math.floor(lv / 4)
              + Math.floor(STR / 5)
              + DEX
              + Math.floor(LUK / 3);
  } else {
    statusATK = Math.floor(lv / 4)
              + STR
              + Math.floor(DEX / 5)
              + Math.floor(LUK / 3);
  }

  const baseWeaponATK = (player as any).weaponATK ?? player.attack ?? 0;
  const statBonus = isRangedWeapon(weaponType)
    ? Math.floor(baseWeaponATK * (1 + 0.005 * DEX))
    : Math.floor(baseWeaponATK * (1 + 0.005 * STR));

  const refineATK = (player as any).refinementBonus ?? 0;
  const bonusATK  = (player as any).bonusATK ?? 0;

  return statusATK + statBonus + refineATK + bonusATK;
}
```

4.2. Show Min ATK and Max ATK in stat screen.

Because weapon variance exists, ATK is a range, not a fixed number.
Show it as "A ~ B":

```ts
const weaponATKEff = getWeaponATKEffective(player);
const wLv = (player as any).weaponLevel ?? 1;
const varianceRange = Math.floor(0.05 * wLv * weaponATKEff);
const statusATK = getStatusATK(player);

const MinATK = statusATK + (weaponATKEff - varianceRange);
const MaxATK = statusATK + (weaponATKEff + varianceRange);

display: `ATK: ${MinATK} ~ ${MaxATK}`;
```

4.3. On level-up, show which stats affect ATK for this player's weapon.

When the player gains a stat point and hovers or allocates STR/DEX, the tooltip should say:

- If equipped with melee weapon:
  - STR: "+1 ATK per point (primary)"
  - DEX: "+1 ATK per 5 points (secondary)"

- If equipped with ranged weapon:
  - DEX: "+1 ATK per point (primary)"
  - STR: "+1 ATK per 5 points (secondary)"

This teaches the player the melee/ranged distinction naturally through the UI.

--------------------------------
5. ENEMY WEAPONS
--------------------------------

Enemies should also have weaponType set in generateEnemyStats:

- Field monsters, melee: weaponType = "none" or "sword" (default melee)
- Ranged monsters (archers, gunners): weaponType = "bow" or "gun"
- Bosses: assign manually per boss design

When enemies don't have weaponType set, getStatusATK defaults to melee formula automatically (safe fallback).

--------------------------------
7. RULE SUMMARY (plain English for reference)
--------------------------------

- Melee weapon equipped:
  - STR = +1 ATK per point (strong).
  - DEX = +1 ATK per 5 points (weak).
  - WeaponATK bonus = 0.5% per STR.

- Ranged weapon equipped:
  - DEX = +1 ATK per point (strong).
  - STR = +1 ATK per 5 points (weak).
  - WeaponATK bonus = 0.5% per DEX.

- LUK: always +1 ATK per 3 points regardless of weapon.
- BaseLv: always +1 ATK per 4 levels regardless of weapon.
- If no weapon equipped: use melee formula as default.
